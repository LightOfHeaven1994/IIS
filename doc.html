Implementace

Stručná dokumentace k implementaci, která popisuje, které části projektu (např. PHP skripty) implementují jednotlivé případy použití.

Pre projekt sme použili Python3 a k nemu framework Flask. Ako databázu sme použili SQLAlchemy. models.py tvorí kostru databázu, a sú tu definované objekty pre entity ako je User, Ticket, Event... . routes.py funguje ako medzivrstva medzi databázou a frontendom. Je rozdelený do ciest pre stránky, ktoré v sebe majú logiku pre manipuláciu dát v databáze. Táto logika je naviazaná na frontend ktorý je tvorený pomocou JS Frameworku Bootstrap a samotného HTML5. Pre prácu s Bootstrapom máme definované rôzne formy v súbori forms.py. Samotné kostry stránok sa nachádzajú v priečinku templates, a dedia štýl od hlavnej stránky layout.html Pri prípade chyby sa aplikácia odkazuje na funkcie v súbore errors.py

Nadštandardná implementácia

Aplikácia podporuje nahrávanie a zobrazovanie obrázkov. Na začiatku má udalosť alebo užívateľ priradený defaultný obrázok ktorý si môže zmeniť. Odkaz na obrázok sa uloží do databázi s pseudonáhodne vygenerovaným menom, a uloží sa do zložky static.

Naďalej je implemtované interaktívne rezervácie stoličiek na udalosti, kde uživateľ vidí ktoré stoličky sú volné, kde v sále sa nachádzajú a môže si ich zarezervovať kliknutím.

Využitie mailového serveru pre obnovenie zabudnutého hesla. Obnova zabudnutého hesla prebieha pomocou odoslania emailu užívatelovi s odkazom v ktorom je vygenerovaný "token". tento token má určitú životnosť po ktorej sa stane neplatným a užívatelovi sa heslo nezmení. Taktiež mailový server využívame pri odosielaní potvrdení o rezervácií stoličiek nezaregistrovaným uživateľom.

Instalace

Stručně popište:
  • Pre server je použitý Virtual Machine od služby Google Cloud, na ktorom sme spustili operačný systém ubuntu. Naďalej sme stiahli a nakonfigurovali nginx webový server , ktorý zbiera trafiku z localhost. Následovne sme nainštalovali server Gunicorn3 ktorý s nginx komunikuje. Potom sme aplikáciu spustili pomocou gunicorn3 run:app v rozhraní tmux ktorý umožňuje mať proces spustený aj po ukončení ssh spojenia.
  • Python==3.6.8
  • Flask==1.1.1
  • Flask-WTF==0.14.2
  • Flask-SQLAlchemy==2.4.1
  • flask-bcrypt==0.7.1
  • flask-login==0.4
  • Pillow==6.2.1
  • flask-mail==0.9.1
  • To run project:
  • 1) Pull repository
  • 2) Create virtual environment by using:
  • a) virtualenv folder name
  • b) python3 -m venv *folder name
  • 3) use command `souce *folder name*/bin/activate` to activate virtual environment
  • 4) use pip install -r requirements.txt for downloading python packages
  • 5) export FLASK_APP=run.py
  • 6) flask run to run application
  • NOTE: for using debug mode use command `export FLASK_DEBUG=1`

Známé problémy

Nginx zachytáva príliš veľké súbory ako sú gif,videá a pod. pri uploadovaní fotky, a teda sa to na serveri nedá vyriešiť lebo nginx vypíše chybu ako prvý.

Nie je implementované vytváranie nových sálov pretože nám neprišlo, že by kino získavalo nové sály moc často keď vôbec. Preto sme sa rozhodli implementovať iné veci.
